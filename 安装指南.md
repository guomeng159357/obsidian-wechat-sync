# Obsidian 微信内容同步助手 — 安装指南

本指南面向**非开发者**，从零开始，一步一步完成安装和配置。

---

## 你需要提前准备的

安装前请确认以下事项：

- 一台 **Mac 电脑**（Windows 类似，但个别命令不同，文末有说明）
- 电脑上已安装 **Obsidian** 桌面版（[下载地址](https://obsidian.md/download)）
- 有一个 **微信账号**，用于关注测试号

其他软件（Node.js、cloudflared 等）下面会一步步教你安装。

预计总耗时：**30-40 分钟**

---

## 第一步：安装 Node.js

Node.js 是整个项目的运行环境。

1. 打开浏览器，访问 [https://nodejs.org](https://nodejs.org)
2. 下载 **LTS 版本**（左侧那个，通常是 20.x 或 22.x）
3. 下载后双击 `.pkg` 文件，一路点"继续"完成安装
4. 安装完成后打开**终端**（在"启动台 → 其他 → 终端"），输入以下命令验证：

```bash
node --version
```

如果显示类似 `v20.11.0` 的版本号，说明安装成功。

---

## 第二步：获取项目代码

我会把整个项目文件夹发给你（通过网盘、U盘等方式），你把它放到电脑上某个位置，比如：

```
/Users/你的用户名/Desktop/Obsidian内容同步助手/
```

然后打开终端，进入这个目录：

```bash
cd /Users/你的用户名/Desktop/Obsidian内容同步助手
```

> **注意**：下面所有命令都在这个目录下执行。可以把终端固定在这个目录。

---

## 第三步：安装项目依赖

在终端中执行（需要几分钟，取决于网络）：

```bash
npm install
```

看到类似 `added 200 packages` 的输出就说明成功了。如果报错，检查网络是否通畅。

---

## 第四步：安装 cloudflared（内网穿透工具）

微信的服务器在公网上，你的电脑在局域网里。cloudflared 的作用是在两者之间搭一座桥。

### 4.1 下载 cloudflared

在终端中逐条执行：

```bash
# 下载（约 20MB，耐心等几分钟）
curl -L "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-darwin-amd64.tgz" -o /tmp/cloudflared.tgz
```

> 如果下载失败，说明 GitHub 连接不稳定。多试几次，或者换网络环境（比如关掉 VPN）。

```bash
# 解压
tar xzf /tmp/cloudflared.tgz
```

```bash
# 赋予执行权限并移动到 bin 目录
chmod +x cloudflared
mkdir -p ~/.local/bin
mv cloudflared ~/.local/bin/cloudflared
```

### 4.2 验证安装

```bash
~/.local/bin/cloudflared --version
```

显示类似 `cloudflared version 2026.6.0` 就说明安装成功。

---

## 第五步：构建 Obsidian 插件

### 5.1 构建

在项目目录下执行：

```bash
cd packages/obsidian-plugin
node esbuild.config.mjs production
```

执行后会在当前目录生成 `main.js` 文件。

### 5.2 安装插件到 Obsidian

1. 打开 Obsidian
2. 点击左下角齿轮图标进入"设置"
3. 找到"第三方插件"，如果"安全模式"是开的，先点"关闭安全模式"
4. 在你的 vault 文件夹中，找到 `.obsidian/plugins/` 目录（如果看不到 `.obsidian` 文件夹，按 `Cmd+Shift+.` 显示隐藏文件）
5. 在 `plugins/` 下创建新文件夹 `wechat-inbox-sync`
6. 把以下三个文件复制进去：
   - `packages/obsidian-plugin/main.js`
   - `packages/obsidian-plugin/manifest.json`
   - `packages/obsidian-plugin/styles.css`
7. 回到 Obsidian，在"第三方插件"列表中找到 **WeChat Inbox Sync**，点击启用

### 5.3 配置插件

启用后在插件右侧点齿轮图标，进入设置：

| 设置项 | 值 |
|--------|-----|
| 收件箱路径 | `/Users/你的用户名/Desktop/Obsidian内容同步助手/sample-inbox` |
| 输出文件夹 | `WeChat`（默认） |
| 附件文件夹 | `WeChat/attachments`（默认） |
| 启用自动同步 | 建议**开启** |
| 自动同步间隔 | `60` 秒 |

> **重要**：收件箱路径必须是**绝对路径**，把"你的用户名"替换成你电脑的实际用户名。不确定用户名的话，终端执行 `whoami` 查看。

## 第六步：申请微信测试号

正式公众号需要认证，但微信提供了免费的测试号用于开发。

1. 浏览器打开：**https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login**
2. 用微信扫码登录
3. 登录后会出现一个测试号管理页面，先**不要关闭**
4. 关注页面上的测试号二维码（用微信扫）

---

## 第七步：启动服务

你需要打开**两个终端窗口**（`Cmd+N` 在终端中新建窗口），两个都要先进入项目目录。

### 终端 A（后端服务）

```bash
cd /Users/你的用户名/Desktop/Obsidian内容同步助手/packages/wechat-backend
export WECHAT_TOKEN=选择一个你自己定的token比如abcd1234
export INBOX_PATH=../../sample-inbox
npx tsx src/server.ts
```

看到以下输出表示启动成功：
```
后端服务已启动: http://localhost:3000/wechat
当前 Token: abcd1234
```

**这个窗口保持打开，不要关！**

### 终端 B（内网穿透）

> **注意**：如果你之前已经开了终端 A 并且关闭了终端（显示 $\sim$），需要先进入目录：
> ```bash
> cd /Users/你的用户名/Desktop/Obsidian内容同步助手
> ```

```bash
~/.local/bin/cloudflared tunnel --url http://localhost:3000
```

看到类似以下输出：
```
Your quick Tunnel has been created!
https://xxxx.trycloudflare.com
```

**记下这个 `https://xxxx.trycloudflare.com` 地址。**

**这个窗口也保持打开，不要关！**

---

## 第八步：配置微信测试号

回到第六步打开的测试号页面：

1. 找到"接口配置信息"区域
2. **URL**：填写 `https://xxxx.trycloudflare.com/wechat`（注意结尾是 `/wechat`，xxxx 替换为你终端 B 显示的地址）
3. **Token**：填写你在终端 A 设置的 Token（如 `abcd1234`）
4. 点击"提交"

如果提示"配置成功"，恭喜！如果没有，检查：
- 两个终端是否都在运行
- Token 是否完全一致
- URL 结尾是否有 `/wechat`

---

## 第九步：测试

### 9.1 微信发消息

用微信给测试号发一条文字消息，比如"今天天气真好"。

### 9.2 检查同步

在 Obsidian 中，看左侧文件列表是否出现 **WeChat** 文件夹，里面是否有刚发的内容对应的笔记。

如果开启了自动同步，最多等 60 秒会自动出现。如果想立即看到，在 Obsidian 中按 `Cmd+P`，输入"立即同步收件箱"，回车。

---

## 第十步（可选）：不启动微信也能测试

如果只是想测试 Obsidian 同步功能，不需要微信，用自带的模拟器：

```bash
cd /Users/你的用户名/Desktop/Obsidian内容同步助手
npx tsx packages/inbox-simulator/src/cli.ts text "这是一条测试消息"
```

然后在 Obsidian 中手动同步。

## 日常使用

### 每次开机后

启动三个东西（可以设置开机自启，但不强制）：

1. 打开 Obsidian
2. 打开终端，启动**终端 A（后端服务）**
3. 打开另一个终端，启动**终端 B（cloudflared）**

### cloudflared 地址变了怎么办

每次启动 cloudflared 都会生成一个新地址。如果地址变了：

1. 记下终端 B 显示的新地址
2. 打开微信测试号页面
3. 更新"接口配置信息"的 URL
4. 无需重启后端

### 修改笔记模板

在 Obsidian → 设置 → WeChat Inbox Sync → 下拉到"笔记模板"，可以直接编辑。

## 故障排查

| 问题 | 解决 |
|------|------|
| `npm install` 报错 | 检查网络，重试。如果持续失败，可能需要换网络环境 |
| cloudflared 下载失败 | GitHub 连接不稳定，多试几次或换个时段 |
| 微信测试号"配置失败" | 1. 确认终端 A 和 B 都在运行 2. 确认 Token 一致 3. 在终端执行 `curl 你的cloudflared地址/wechat` 看能否连通 |
| Obsidian 看不到 WeChat 文件夹 | 1. 确认插件已启用 2. 检查收件箱路径是否正确 3. 执行"立即同步收件箱"命令 |
| 微信发送后没有笔记 | 1. 检查终端 A 是否有日志输出 2. 检查 sample-inbox/pending/ 有没有新 JSON 文件 3. Obsidian 中手动点同步 |

## Windows 用户注意事项

如果你或你的朋友使用 Windows，大部分流程相同，有少量差别：

- 路径用 `\` 而不是 `/`，比如 `C:\Users\用户名\Desktop\Obsidian内容同步助手\`
- cloudflared 下载链接中的 `darwin-amd64` 改为 `windows-amd64`
- 环境变量设置用 `set WECHAT_TOKEN=abc123` 而不是 `export`
- Obsidian vault 的 `.obsidian` 文件夹可能在 `C:\Users\用户名\Documents\Obsidian\`
- 终端用"命令提示符"或 PowerShell
